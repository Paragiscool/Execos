# State Machine Diagram

Because LLMs are non-deterministic, EXECOS relies on a strictly deterministic finite state machine (`ExecutionContext`).

## Task State Transitions

```text
       ┌───────────┐
       │   DRAFT   │ (Generated by LLM, pre-validation)
       └─────┬─────┘
             │ (Policy Approved)
             ↓
       ┌───────────┐
       │ SCHEDULED │ (Locked into Execution Calendar)
       └─────┬─────┘
             │
      ┌──────┴──────┐
      ↓             ↓
┌───────────┐ ┌───────────┐
│ IN_FLIGHT │ │  MISSED   │ (Triggers Replanning)
└─────┬─────┘ └─────┬─────┘
      │             │
      ↓             ↓
┌───────────┐ ┌───────────┐
│ COMPLETED │ │ CANCELLED │
└───────────┘ └───────────┘
```

When a task transitions to `MISSED`, the Execution Core emits a `MissedBlock` event. The Replay Engine immediately halts, the Digital Twin is updated, and the Planner is re-invoked to generate a delta-plan.
